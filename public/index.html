What can the database do?

The purpose of NC-News is to set up a backend with an API, that can connect to our frontend for testing and practice purposes.

Due to that, not all functionality is currently available and will be added in the future.

Requests Available

So far, there is functionality to get topics, articles and users as full tables. To do this you can call /api/:table_name

You can also get a specific article by choosing a number from 1-15, and see comments on that article eg /api/articles/3/comments

You can post a comment to a specific article with /api/articles/:article_id/comments. The request will accept an object with keys of username: varchar and body: varchar. Must be a valid username to comment.

Patch can be used to change the amount of votes on an article using /api/articles/:article_id. The request will accept an object with a key of inc_votes: INT

Articles

This table has the following columns:

  1. article_id
  2. title
  3. topic
  4. author
  5. body
  6. created_at
  7. votes
  8. article_img_url

Comments

This table has the following columns:

  1. comment_id
  2. article_id
  3. body
  4. votes
  5. author
  6. created_at

Topics

This table has the following columns:

  1. description
  2. slug
  3. img_url

Users

This table has the following columns:

  1. username
  2. name
  3. avatar_url

Error Handling

NC-News uses conventional http error handling and should give a human readable response.
